# 빠른 시작 (한국어)

코딩 지식 없어도 됩니다. 처음 세팅은 20~30분. 나머지는 쓰면서 저절로 채워집니다.

## 0. 준비물

- [클로드 코드](https://claude.com/claude-code) 설치
- 이 워크스페이스를 둘 폴더 (아이클라우드·드롭박스·구글 드라이브 같은 클라우드 동기화
  폴더를 추천 — 여러 기기에서 에이전트 기억이 따라옵니다)

## 1. 내 사본 만들고 열기

권장: 깃허브 페이지에서 **"Use this template" → PRIVATE(비공개) 저장소 생성** 후 그걸
복제하세요. 이 워크스페이스엔 곧 당신의 인생이 담기므로 비공개가 기본값입니다. 그냥
구경만 할 거면 원본을 직접 복제해도 됩니다.

```bash
git clone https://github.com/YOUR-USERNAME/my-workspace.git my-workspace
cd my-workspace
claude
```

`claude`를 실행하면 채팅 프롬프트(`>`)가 나타납니다. 여기에 평소 말하듯 **한국어로**
입력하면 됩니다. 그게 인터페이스의 전부입니다.

가장 빠른 길 — 한 줄 붙여넣기:

> **"INSTALL.md 읽고 이 워크스페이스 세팅해줘."**

에이전트가 bootstrap(개인 파일 생성) → 온보딩 인터뷰 → 훅 연결 제안까지 알아서 합니다.

### 1.5. 개인 파일 생성 (직접 할 때)

```bash
bash tools/bootstrap.sh
```

`*.example` 시드를 실제 파일명으로 복사합니다(`character.md.example` → `character.md`).
이 실제 파일들은 **git 추적 제외**라 당신 데이터는 기본적으로 비공개 — 포크를 공개해도
새어나가지 않습니다. 온보딩 에이전트가 이걸 대신 실행해 줍니다.

### 이 파일들 어떻게 편집하나요? (쉬운 순서로 세 가지)

코딩 몰라도 됩니다. 편한 걸 고르세요:

1. **그냥 에이전트에게 부탁** (가장 쉬움). 클로드 코드에서: *"CLAUDE.md 열어서 내 이름,
   시간대, 언어 채워줘. 값은 내가 불러줄게."* 에이전트가 대신 편집합니다.
2. **아무 텍스트 에디터.** 전부 일반 텍스트 파일입니다. VS Code로 폴더를 열거나,
   텍스트편집기/메모장으로 파일 하나를 열어 수정 후 저장.
3. **터미널**이 편하면: `nano CLAUDE.md`.

### 안전한가요? 되돌릴 수 있나요?

네. 안전장치 두 가지:
- 클로드 코드는 **변경할 때마다 보여주고 승인을 받습니다**(끄지 않는 한). 몰래 바뀌는 일
  없습니다.
- 이 폴더는 **git 저장소**라서 모든 버전이 저장됩니다. 뭔가 잘못되면 에이전트에게
  *"방금 변경 git으로 되돌려줘"*라고 하면 됩니다. git을 직접 배울 필요 없습니다.

## 2. 에이전트가 직접 세팅해주게 하기

가장 빠른 길은 그냥 부탁하는 겁니다:

> "CLAUDE.md랑 docs/ARCHITECTURE.md 읽고, 플레이스홀더 채우는 걸 도와줘.
> 한 번에 하나씩 질문하면서 인터뷰해줘."

`CLAUDE.md`의 `{{OWNER_NAME}}`, `{{TIMEZONE}}`, `{{PRIMARY_LANGUAGE}}`를 채우고
정체성 파일 초안까지 같이 잡아줍니다. 직접 하고 싶으면 아래를 따라가세요.

## 3. 중심부터 채우기 (`00_SELF`)

> **오늘 최소한만:** `character.md`에 솔직한 5줄만 쓰세요. 그게 다입니다. 여기서 멈추고
> 바로 쓰기 시작해도 됩니다. 아래는 전부 "쓰면서 채우는" 것들입니다.

첫날은 이 부분만 중요합니다. 나머지는 선택입니다.

1. **`00_SELF/identity/character.md`** — 당신이 누구인지. 완벽하게 쓰려고 하지 마세요.
   솔직한 다섯 줄이 완벽한 에세이보다 낫습니다. 많이 써도 괜찮습니다 — 에이전트는 분량이
   많아서가 아니라 *모순*이 있을 때 헷갈립니다. 일관되게만 쓰세요.
2. **`00_SELF/people/people.md`** — *한 번에 다 채우지 말고, 사람이 등장할 때 추가하세요.*
   오늘 채울 필요 **없습니다.** 비워둬도 됩니다 — 새 사람이 처음 나오면 에이전트가 "누구
   신가요?" 물어보고, 그때 알려주면 됩니다. 추가할 때만 파일 맨 위 형식을 따르세요.
3. **`00_SELF/memory/agent_brain/USER.md`** — 당신이 선호하는 소통 방식 (말투·길이·
   싫어하는 것).

`character_ai_mirror.md`, `decisions_log.md`, `feedback_patterns.md`는 거의 비워두세요.
쓰다 보면 에이전트가 채웁니다.

## 4. 자동 로딩 훅 연결 (선택 — 어려우면 건너뛰기)

> **코딩 모르면 이 단계 통째로 건너뛰세요.** 이거 없어도 워크스페이스는 완전히 작동합니다 —
> 세션 시작 때 "내 shared_memory랑 todo 읽어줘"라고 말하면 되고, 훅은 그걸 자동으로 해줄
> 뿐입니다. 설정 파일 편집이 편해지면 그때 돌아오세요.

두 개의 훅이 워크스페이스를 살아있게 만듭니다: 최근 맥락 자동 로드(SessionStart) +
답변 게이트 핵심 규칙("개인 맥락 → 페르소나 OFF")을 강제하는 컨텍스트 게이트
(UserPromptSubmit). JSON 직접 편집 대신 인스톨러를 쓰세요 — 기존 설정을 **덮어쓰지 않고
병합**합니다(멱등, 재실행 무해):

```bash
python3 tools/install_hooks.py --dry-run   # 미리보기
python3 tools/install_hooks.py             # 적용 (.bak 백업 생성)
```

아니면 에이전트에게 *"내 워크스페이스 훅 설치해줘"*. 컨텍스트 게이트 쓰기 전
`tools/hooks/context_gate.py`의 `KEYWORDS`를 당신 언어로 바꾸세요(파일에 한국어 예시 있음).
SessionStart 훅은 상대경로라 워크스페이스 루트에서 클로드 코드를 여세요.

## 5. 그냥 쓰기 시작

에이전트에게 말을 걸면 됩니다. 쓰다 보면:
- 확정한 결정 → `decisions_log.md`에 기록됨
- 거절한 것 → 이유가 `feedback_patterns.md`에 저장됨 (다음엔 그 제안 안 함)
- 당신에게서 2번 이상 반복 관찰된 패턴 → `character_ai_mirror.md`
- 마감 있는 프로젝트 → `10_WORK/`에 폴더 (`_PROJECT_TEMPLATE/` 복사)
- 마감 없는 역할 → `20_DOMAINS/`에 폴더 (`_DOMAIN_TEMPLATE/` 복사)

## 6. 데이터는 기본적으로 비공개

**여기서 할 일 없습니다.** 이 템플릿은 개인 파일을 `*.example` 시드로 배포하고,
`bootstrap.sh`가 그걸 **git 추적 제외**된 실제 파일명으로 복사합니다. 그래서 정체성·인물·
결정·기억은 애초에 추적되지 않고 공개되지도 않습니다 — 사본이 public이어도요. 프레임워크
파일만 추적되므로 데이터 손실 없이 업데이트를 당길 수 있습니다([UPDATE.md](../UPDATE.md)).

그래도 지킬 습관 둘: ① 그래도 **비공개 저장소를 권장**(인생을 굳이 공개 URL에 둘 이유
없음) ② 진짜 비밀 값은 `.env`(이미 추적 제외)에만, 추적되는 `.md`엔 금지. 개인 데이터가
**한 번이라도 공개 푸시된 적 있다면** 유출로 간주하세요 — 나중에 비공개로 바꿔도
포크·캐시는 안 지워집니다.

## 선택: 격리된 두 번째 에이전트 (`90_SHARED`)

파트너·가족과 공유하는 에이전트 — 공유 계획은 보되 당신의 개인 `00_SELF`는 **건드리지
않는** — 가 필요하면 `90_SHARED/`가 그 용도입니다. 단, 이 폴더 `CLAUDE.md`의 규칙은
지시문이지 접근 제어가 아닙니다: 진짜 격리가 필요하면 클로드 코드 세션을 `90_SHARED/`
**안에서** 시작하거나(상위 폴더가 아예 안 보임) 샌드박스로 실행하세요. 필요 없으면
폴더째 삭제하면 됩니다.
