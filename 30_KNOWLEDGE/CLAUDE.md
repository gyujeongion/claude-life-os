# 30_KNOWLEDGE — research & knowledge

> What you've learned and what the agent has researched for you. This is `[AI]` territory:
> analysis, research output, reference material. Keep it separate from `[OWNER]` decisions
> in `00_SELF/decisions/`.
>
> 배운 것과 에이전트가 리서치한 것. `[AI]` 영역: 분석·리서치 결과·참고자료.
> `00_SELF/decisions/`의 `[OWNER]` 결정과 분리.

## Suggested layout

```
30_KNOWLEDGE/
├── research/     # deep-dives, analysis the agent produced (working material, [AI])
├── documents/    # finished reports you actually read (with a .md summary + an INDEX)
└── reference/    # durable reference notes
```

## Rules

- Tag research `[AI]`. It informs decisions; it doesn't make them.
- When the agent produces a finished deliverable you'll actually read, keep three things
  together: the file, a short `.md` summary (so it's searchable), and a one-line entry in
  an `INDEX.md`. A PDF alone isn't findable later; the summary is.
- Analysis docs and proposal/plan docs stay separate.
